## 图表规范

| 数据 | 图表类型 | 参数 |
|------|---------|------|
| $APPEALS 8维度 | 玫瑰图(roseType:radius) | radius:['30%','70%']，8个维度 |
| 满意度鸿沟 | 雷达图(双系列) | 用户期望(蓝) vs 竞品实测(红)，半透明红色填充差距区 |
| ABSA方面级情感 | 水平堆叠条形图 | 正(绿)/负(红)/中(灰)，危机项标红边框 |
| 用户旅程摩擦 | 水平折线/阶梯图 | 6阶段×摩擦分(0-10)，峰值标注红点，终值标注旗帜 |
| KANO机会分 | 气泡图 | x=频率, y=机会分, 气泡大小=severity |
| 市场垄断度 | 饼图 | Top3占比 |
| 价格分布 | 柱状图 | 价格段×销量 |

## 新增模块视觉规范

### Module 3 — 用户画像卡（PSPS）
- 布局：4列网格，每列一个维度（P/S/P/S）
- 卡片顶部：维度名称（大写字母）+ 图标（👤/🎬/😤/💡）
- 卡片颜色：Persona=#e3f2fd / Scenario=#f3e5f5 / Pain=#fce4ec / Solution=#e8f5e9
- Solution Gap 单独展示为一条横向「电梯演讲」条带，背景 #1b5e20，白色文字

### Module 5 — 方面级情感（ABSA）
- 水平堆叠条形图，每行一个方面
- 正面：#4caf50 / 负面：#f44336 / 中性：#9e9e9e
- crisis_flag=true 的行：左边框 3px solid #b71c1c，背景 #fff8f8，右侧显示 ⚠️ 标签

### Module 7 — 痛爽痒三维图谱
- 三列卡片，各列顶部图标：😣痛点 / 😊爽点 / ✨痒点
- 每列：强度评分（大数字）+ 具体 items 列表 + 底部营销文案建议（斜体，灰色背景）
- 痛点列：左边框 #b71c1c / 爽点列：左边框 #1565c0 / 痒点列：左边框 #7b1fa2

### Module 8 — 用户旅程摩擦图
- 上方：6阶段折线/阶梯图（ECharts），peak_or_end=true 的点标红色大圆点
- 下方：Top3 摩擦阶段卡片（横排），各卡片显示阶段名/摩擦分/核心改善建议
- 底部：峰终洞察文本条带（背景 #e8eaf6）
- 3条改善建议：绿色左边框列表

### Module 10 — 差评原声剧场（分区版）
- 按 category 分三区展示：
  - 😣 痛点区：红色标题栏
  - 😊 爽点区：蓝色标题栏（来自3星含正面内容的评论）
  - ✨ 痒点区：紫色标题栏
- 每区最多3条，共最多9条

## 评论卡片规范

- 布局：`grid-template-columns: repeat(auto-fill, minmax(280px, 1fr))`
- 每卡：星级(橙色) + 跨站标签(蓝色) + 标题(粗体13px) + 正文(13px,截280字) + 日期(11px,灰色)
- 最多显示9条

## 采集说明模块

- 达标(≥100条)：绿色背景 `#e8f5e9`，绿色左边框
- 不达标(<100条)：黄色背景 `#fff8e1`，橙色左边框，附红色说明文字

## 身份标签规则

| 条件 | 标签 |
|------|------|
| 评分≥4.5 且 评论数≥1000 | `[市场统治者]` 红色 |
| 评分≥4.8 | `[口碑天花板]` 深红色 |
| 其他 | `[普通竞品]` 灰色 |

## 文件大小校验

- 生成报告 HTML < 50KB → 视为不完整，重新生成
- 正常报告大小：50KB - 300KB
